Instructions for Game Play

A linked list is a linear collection of data elements, whose order is not given by their physical placement but instead each element points to the next.
Ready to learn about Linked Lists? Here are the rules:
  • This is a Linked List educational game that closely resembles the popular game of BrickBreaker.
  • The object of the game is to properly construct a Linked List by hitting the bricks (nodes) in the correct order using the ball and paddle.
  • First, initialize the list by setting head = null by first hittting the "head" brick then the "null" brick.
  • Once the list is initialized, you will be prompted to insert random nodes into the list.
  • Insert the nodes by correctly traversing the list by hitting the nodes with the ball. Start with Head and connect the nodes accordingly.
  • Use the arrrow keys to control paddle and WAD keys to directionally release the ball.
  • You will have 5 lives to complete the list, and a life will be lost every time the ball hits the bottom.
Good Luck!

Ready to Play? Press the A/W/D Key's for directional shooting.

Why Linked Lists?



Dynamic Size & Ease of Insertion/Deletion

Linked lists store linear data like arrays do. However arrays are limited because their size is fixed. One must know the upper limit on the number of elements in advance, and inserting a new element in an array of elements is expensive because room has to be created for the new elements, and to create room for existing elements that have shifted.
